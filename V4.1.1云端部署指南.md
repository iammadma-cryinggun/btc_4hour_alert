## 部署步骤


### 步骤2: 配置代理（可选）
如果服务器需要代理访问Binance/Coinalyze：

编辑脚本中的代理配置：
```python
# 第173-177行
self.proxy_enabled = True   # 改为True
self.proxy_host = "127.0.0.1"  # 改为代理IP

```

### 步骤3: 测试运行
```bash

python3 BTC4小时策略预警v4.1.1_双信号版.py

# 按Ctrl+C停止
```

### 步骤4: 后台运行（使用systemd）
创建systemd服务文件：

```bash
sudo nano /etc/systemd/system/btc-physics-v411.service
```

内容：
```ini
[Unit]
Description=BTC Physics Signal System V4.1.1
After=network.target

[Service]
Type=simple
User=你的用户名
WorkingDirectory=/home/你的用户名/btc-physics-v411
ExecStart=/usr/bin/python3 /home/你的用户名/btc-physics-v411/BTC4小时策略预警v4.1.1_双信号版.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启用并启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable btc-physics-v411.service
sudo systemctl start btc-physics-v411.service
```

查看服务状态：
```bash
sudo systemctl status btc-physics-v411.service
```

查看日志：
```bash
# 实时日志
tail -f physics_alert_v3_1_complete.log

# systemd日志
sudo journalctl -u btc-physics-v411.service -f
```

### 定期备份脚本
```bash
#!/bin/bash
# backup.sh - 每周备份重要数据

BACKUP_DIR="/home/你的用户名/btc-physics-backup"
DATE=$(date +%Y%m%d)

mkdir -p $BACKUP_DIR

# 备份数据文件
cp basic_physics_signals.json $BACKUP_DIR/basic_physics_signals_$DATE.json
cp trade_history.json $BACKUP_DIR/trade_history_$DATE.json

# 保留最近8周的备份
find $BACKUP_DIR -name "*.json" -mtime +56 -delete

echo "备份完成: $DATE"
```

添加到crontab（每周日凌晨3点执行）：
```bash
crontab -e

# 添加这一行
0 3 * * 0 /home/你的用户名/btc-physics-v411/backup.sh
```

## 常见问题

### Q1: 程序启动后没有信号
A: 检查以下几点：
1. 是否能访问Binance API
2. 是否能访问Coinalyze API（需要代理）
3. 查看日志文件 `physics_alert_v3_1_complete.log`

### Q2: Telegram收不到消息
A: 检查：
1. `telegram_token` 是否正确
2. `telegram_chat_id` 是否正确
3. 服务器能否访问Telegram API

### Q3: 基础信号和交易信号有什么区别
A:
- **基础信号**：验证4原始计算，每4H记录，仅用于复盘
- **交易信号**：V4.1 Smart Ape系统，带EMA21入场和各种过滤，实际交易

### Q4: 如何查看程序是否正常运行
A:
```bash
# 查看进程
ps aux | grep BTC4小时策略预警

# 查看最新日志
tail -20 physics_alert_v3_1_complete.log

# 查看基础信号记录
tail -10 basic_physics_signals.json
```

### Q5: 程序崩溃后如何重启
A:
```bash
# 如果使用systemd
sudo systemctl restart btc-physics-v411.service

# 如果使用nohup
# 先查看进程ID
ps aux | grep BTC4小时策略预警
# 杀死进程
kill <进程ID>
# 重新启动
nohup python3 BTC4小时策略预警v4.1.1_双信号版.py > output.log 2>&1 &
```

可以创建Python脚本来分析基础信号：
```python
import json
import pandas as pd

# 加载信号
with open('basic_physics_signals.json', 'r') as f:
    signals = json.load(f)

# 转换为DataFrame
df = pd.DataFrame(signals)

# 分析
print(f"总信号数: {len(df)}")
print(f"看空奇点: {len(df[df['regime_type']=='BEARISH_SINGULARITY'])}")
print(f"看涨奇点: {len(df[df['regime_type']=='BULLISH_SINGULARITY'])}")

# 保存为CSV
df.to_csv('basic_physics_signals_analysis.csv', index=False)
```
