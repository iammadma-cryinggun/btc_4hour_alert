## 核心更新

### 1. 双信号系统架构

V4.1.1同时运行两套独立的信号系统：

#### 【基础物理信号】（新增）
- **计算频率**: 每4小时（与4H K线收盘对齐）
- **信号类型**: 验证4原始物理信号
- **功能**: 纯记录，不参与交易
- **保存位置**: `basic_physics_signals.json`
- **通知方式**: Telegram（仅SINGULARITY信号）
- **用途**: 未来复盘分析，对比基础信号vs过滤后信号的表现

#### 【V4.1 Smart Ape信号】（保留）
- **计算频率**: 每4小时（信号产生） + 每1分钟（EMA21入场检查）
- **信号类型**: 经过过滤的高级信号
- **功能**: 实际交易执行
- **过滤机制**:
  - SKIP过滤（LS>3.5, FR<-2%）
  - 黄金阈值过滤（空单LS<2.0）
  - EMA21入场过滤
  - Smart Ape动态风控
- **用途**: 真实交易

### 3. 新增代码类

#### BasicPhysicsSignalTracker
- 位置: 第132-203行
- 功能: 追踪和保存基础物理信号
- 方法:
  - `add_signal()` - 添加新信号
  - `get_recent_signals()` - 获取最近N小时信号
  - `print_summary()` - 打印信号统计

#### scheduled_basic_physics_signal_check()
- 位置: 第3899-3975行
- 功能: 每4H定时计算基础信号
- 触发时间: 00:00:10, 04:00:10, 08:00:10, 12:00:10, 16:00:10, 20:00:10

## 升级指南

### 从V4.1升级到V4.1.1

1. **停止V4.1程序**
   ```bash
   # 如果使用systemd
   sudo systemctl stop btc-physics-v41.service
   ```

2. **备份现有数据**
   ```bash
   cp trade_history.json trade_history.json.backup
   cp position_tracker_aligned_status.json position_tracker_aligned_status.json.backup
   ```

3. **上传V4.1.1文件**
   ```bash
   # 上传 BTC4小时策略预警v4.1.1_双信号版.py
   ```

4. **保留状态文件**（不要删除！）
   - `position_tracker_aligned_status.json`
   - `battle_aligned_status.json`
   - `trade_history.json`
   - `dxy_data.csv`

5. **启动V4.1.1**
   ```bash
   # systemd方式
   sudo systemctl start btc-physics-v411.service

   # 或者直接运行
   nohup python3 BTC4小时策略预警v4.1.1_双信号版.py > output.log 2>&1 &
   ```

6. **验证运行**
   ```bash
   # 检查进程
   ps aux | grep v4.1.1

   # 查看日志
   tail -f physics_alert_v3_1_complete.log
   ```

## 常见问题

### Q: 基础信号会参与交易吗？
A: 不会。基础信号只记录，不参与交易。V4.1 Smart Ape系统独立运行。

### Q: 可以关闭基础信号记录吗？
A: 可以。注释掉第4020-4023行的定时任务即可。

### Q: 基础信号文件会很大吗？
A: 不会。每天6条记录，每条约300字节，一年约650KB。

### Q: 如何分析基础信号数据？
A: 导出JSON文件，用pandas读取分析。参考"云端部署指南"中的分析工具章节。
